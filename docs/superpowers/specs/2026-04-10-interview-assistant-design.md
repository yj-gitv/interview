# 面试助手（Interview Assistant）设计规格

## 概述

一款本地运行的 Web 面试辅助工具，帮助面试官完成从简历筛选到面试总结的全流程。通过系统音频捕获实现腾讯会议面试的实时转录和 AI 辅助，在保护候选人隐私的前提下提升面试效率和评估质量。

**目标用户**：个人面试官，独立使用，每周面试 5+ 人
**岗位类型**：非技术岗为主（产品、运营、市场、设计等）

## 整体架构

三层架构：

### 用户交互层

- **浏览器 Web UI**：面试官的主操作界面，运行在 localhost
- **腾讯会议**：线上面试进行，音频通过 BlackHole 路由到系统
- **钉钉 / 飞书**：接收 HR 发来的简历，推送面试结果摘要

### 本地核心服务层（Python FastAPI）

- 简历解析（PDF/Word）
- PII 脱敏层
- 匹配打分（JD ↔ 简历）
- 面试问题生成
- 实时面试辅助
- 候选人回答评估
- 面试总结报告生成

### 基础设施层

- **公司 OpenAI 网关**（兼容 OpenAI API 格式）：LLM 分析，仅接收脱敏后的文本
- **本地 faster-whisper**：语音转文字，音频数据完全不出机器
- **本地 SQLite + SQLCipher**：候选人数据 AES-256 加密存储

### 关键数据流

腾讯会议音频 → BlackHole 虚拟音频 → 本地 faster-whisper 转录 → PII 脱敏 → 公司 OpenAI 网关分析 → 结果存入本地加密数据库

## 技术栈


| 层面    | 技术选择                              | 理由                                           |
| ----- | --------------------------------- | -------------------------------------------- |
| 后端    | Python + FastAPI                  | AI/ML 生态好，Whisper/OpenAI SDK 原生支持，异步性能优秀     |
| 前端    | React + TypeScript + Tailwind CSS | 组件化开发，实时 WebSocket 更新体验好                     |
| 语音转录  | faster-whisper（本地）                | CTranslate2 加速版 Whisper，macOS 实时转录性能够用，音频零外泄 |
| LLM   | 公司 OpenAI 网关                      | 兼容 OpenAI API 格式，支持多模型切换                     |
| 数据库   | SQLite + SQLCipher                | 轻量本地数据库，AES-256 透明加密                         |
| 音频捕获  | BlackHole（虚拟音频驱动）                 | macOS 成熟方案，零侵入                               |
| 简历解析  | PyMuPDF + python-docx             | 支持 PDF 和 Word 两种主流简历格式                       |
| 实时通信  | WebSocket                         | 面试过程中转录和建议实时推送                               |
| IM 集成 | 钉钉/飞书 Webhook + 机器人 API           | 轻量集成，无需复杂审批                                  |


### LLM 模型策略

架构上模型可配置，不绑定单一模型，可在设置页面为不同功能指定不同模型：

- **实时追问建议**：使用快速模型（如 GPT-4o-mini），优先低延迟
- **简历匹配打分 / 面试总结**：使用强能力模型（如 GPT-4o），质量优先

## 核心功能模块

### 模块一：简历匹配度打分

**输入**：JD + 候选人简历（PDF/Word）+ 面试官个人偏好（可选）

**流程**：

1. 解析简历，提取结构化信息（工作经历、项目经验、技能、教育背景等）
2. PII 脱敏（姓名→代号，手机号/邮箱/身份证号→移除）
3. 脱敏简历 + JD + 偏好发送给 LLM 多维度评估
4. 返回结构化评分

**评分维度**（非技术岗）：

- 岗位经验匹配度（0-100）
- 行业背景相关性（0-100）
- 核心能力匹配度（0-100）
- 成长潜力评估（0-100）
- 综合推荐指数（0-100）+ 推荐/待定/不推荐

每个维度附简短说明。同时标注简历中的亮点和风险点（如频繁跳槽、经历断层等）。

### 模块二：面试前问题准备

**输入**：JD + 脱敏简历 + 匹配打分结果 + 面试官偏好/关注点

**输出**：结构化面试问题清单——

1. **开场问题**（1-2 个）：让候选人放松，了解基本情况
2. **经历验证问题**（3-5 个）：STAR 法提问，验证简历关键经历的真实性和深度
3. **能力考察问题**（3-5 个）：根据 JD 核心能力设计情景题
4. **风险探测问题**（1-3 个）：针对匹配打分中发现的风险点
5. **文化匹配问题**（1-2 个）：价值观和团队适配性
6. **候选人提问环节提示**

每个问题附带：考察目的、优秀回答应包含的要素、红旗信号。

面试官可在面试前预审、增删、排序问题清单。

### 模块三：面试实时辅助

**工作方式**：BlackHole 捕获音频 → faster-whisper 实时转录 → LLM 分析 → WebSocket 推送

**实时辅助流程六步链路**：

1. **音频捕获**：腾讯会议 → BlackHole → PyAudio（配置多输出设备，面试官正常听到对话）
2. **本地语音转录**：faster-whisper 每 3-5 秒分段 + VAD 自动分句，完全本地
3. **说话人区分**：利用音频通道分离——面试官麦克风为一路，BlackHole 捕获的候选人远端音频为另一路，天然区分双方发言。作为补充，可用 pyannote-audio 做本地说话人分割以处理串音情况
4. **实时 PII 脱敏**：过滤候选人口述的个人信息
5. **LLM 分析**（快速模型）：问题自动识别、回答要素检查、追问建议生成、即时评分
6. **WebSocket 推送**：实时更新面试辅助面板

**全链路延迟目标 < 5 秒**（Whisper ~3s + LLM ~1.5s）

**问题自动跟踪**：

- 系统通过转录内容语义匹配自动识别当前问题（不要求逐字照念）
- 自动高亮进行中问题，展示要素 checklist
- 话题切换时自动标记上一问题完成并给出即时评分
- 面试官可单击问题列表手动切换（保底）
- 聊到清单外话题时进入"自由对话"模式

**追问建议交互**：

- 新建议以卡片形式滑入右侧建议区顶部
- 旧建议自然向下滚动，离开核心视野但不消失
- 所有建议保留在可滚动列表中，随时可翻回
- 历史建议变为次要样式（字体略小、颜色略淡）但完全可读
- 触发条件：回答缺具体数据/案例、与简历有出入、过于简短、提到有价值新话题

### 模块四：候选人回答评估

**两种模式**：

- **实时评估**：每个问题回答完自动简短评分（好/一般/差 + 一句话理由）
- **面后深度评估**：基于完整转录做全面分析

**评估维度**（非技术岗）：

- 表达清晰度：逻辑是否清楚、言之有物
- 案例丰富度：是否用具体案例支撑观点
- 思维深度：分析问题的层次和全面性
- 自我认知：对自身优劣势的真实认知
- 岗位热情：对岗位和公司的了解和兴趣

### 模块五：面试总结

**输入**：完整面试转录 + 实时评估数据 + 简历匹配结果

**输出**：结构化面试报告——

1. 候选人概要（一段话总结）
2. 各维度评分（雷达图可视化）
3. 关键发现：亮点和顾虑点各 3-5 条
4. 与 JD 匹配分析：逐项对照 JD 要求的达成情况
5. 录用建议：推荐/待定/不推荐 + 理由
6. 后续建议：下一轮重点考察方向

报告可导出 PDF，可推送摘要到钉钉/飞书。

## UI 布局

### 面试实时辅助面板（三栏布局）

- **左栏（实时转录）**：面试官发言蓝色标记，候选人发言橙色标记，自动滚动
- **中栏（问题清单 & 要素检查）**：当前问题高亮蓝色边框，要素自动打勾，已完成问题灰化，待提问题列在下方
- **右栏（追问建议）**：新建议顶部高亮，历史建议向下滚动保留可翻阅
- **顶部状态栏**：候选人代号（脱敏）、岗位、录音状态、面试时长、结束按钮

## 隐私保护方案

### PII 脱敏层

所有数据发送给 LLM 前的必经关卡：


| 信息类型 | 脱敏方式         |
| ---- | ------------ |
| 姓名   | 替换为代号（候选人 A） |
| 手机号  | 完全移除         |
| 邮箱   | 完全移除         |
| 身份证号 | 完全移除         |
| 家庭住址 | 完全移除         |
| 公司名称 | 保留（对评估有价值）   |
| 学校名称 | 保留（同上）       |


**实现方式**：

- 正则匹配：手机号、身份证号、邮箱等格式化信息
- 本地 NER 模型（优先 HanLP，中文实体识别准确率高且支持离线）：人名、地址等非格式化信息
- 脱敏映射表本地存储，面试报告展示时可还原

### 本地数据安全

- SQLCipher AES-256 加密，启动时需输入解锁密码
- 简历原文件和录音存在本地加密目录
- 无远程同步，数据只存本机
- 可配置自动清理策略（如面试完成 90 天后删除）

### 传输安全

- 公司 OpenAI 网关请求走 HTTPS
- Web UI 运行在 localhost，不暴露到网络
- 钉钉/飞书推送只包含结论性信息（推荐等级 + 一句话），不含隐私

## 数据模型

### 核心实体

**岗位 (Position)**：独立管理的一级实体

- JD 原文、岗位名称、部门、核心能力列表
- 面试官个人偏好备注
- 状态：招聘中 / 已关闭 / 暂停
- 支持复用：同一岗位关联多个候选人，无需重复导入 JD
- 常用岗位置顶，按最近使用排序

**候选人 (Candidate)**

- 原始简历文件（加密存储）
- 解析后结构化信息
- PII 字段单独加密，脱敏版本用于 AI 分析
- 候选人代号

**简历匹配 (ResumeMatch)**

- 关联岗位 + 候选人
- 各维度评分 + 说明
- 亮点列表、风险点列表
- 综合推荐等级

**面试场次 (Interview)**

- 关联岗位 + 候选人
- 面试时间、时长、状态（待面试 / 进行中 / 已完成）
- 问题清单

**面试转录 (Transcript)**

- 关联面试场次
- 时间戳 + 说话人 + 转录文本（原始版 + 脱敏版）
- 分段存储

**回答评估 (Evaluation)**

- 关联面试场次 + 对应问题
- 各维度评分 + AI 评语 + 面试官手动备注

**面试总结 (Summary)**

- 关联面试场次
- 综合评分、关键发现、录用建议
- PDF 导出路径、钉钉/飞书推送状态

### 实体关系

一个岗位 → 多个候选人 → 每个候选人有一次匹配打分 → 每次面试产生一份转录 + 多条评估 + 一份总结。同岗位候选人自然归组，支持并排对比。

## 钉钉/飞书集成

轻量 Webhook 方案：

**接收简历（入方向）**：

- 钉钉：创建企业内部应用（机器人），配置事件订阅回调 URL 指向本地后端，HR 在群里 @机器人 发送简历文件时触发回调
- 飞书：创建自建应用（机器人），配置事件订阅，HR 发送简历文件时通过事件推送到后端
- 后端接收事件后自动下载文件、解析、创建候选人记录
- Web UI 手动上传作为备选（无需配置 IM 也能正常使用）

**推送结果（出方向）**：

- 面试总结完成后，通过 Incoming Webhook 推送摘要到指定群或个人
- 推送内容：候选人代号 + 岗位 + 推荐等级 + 一句话总结（无隐私信息）

**配置**：设置页面填入 Webhook 地址，钉钉和飞书二选一或都配。

## 完整使用流程

### 阶段一：面试前准备（面试前 1-2 天）

1. 创建或选择已有岗位 JD（同一 JD 只需录入一次）
2. 导入简历（钉钉/飞书自动接收或手动上传）
3. 自动匹配打分（各维度分数、亮点、风险）
4. 生成面试问题清单（可预审调整）

### 阶段二：面试进行中（30-60 分钟）

1. 启动面试模式，连接 BlackHole 音频源
2. 音频通道自动分离（面试官麦克风 vs 候选人远端音频）
3. 三栏实时辅助面板，基本零操作
4. 结束面试

### 阶段三：面试后总结（结束后 1-2 分钟自动生成）

1. AI 自动生成结构化报告
2. 审阅、补充个人备注
3. 导出 PDF + 推送摘要到钉钉/飞书
4. 同岗位候选人并排对比

## 边界情况处理

- **网络中断**：本地 Whisper 不受影响；AI 分析暂停，恢复后自动补充
- **音频问题**：BlackHole 断开时提示异常，切换到手动输入模式
- **候选人说外语/方言**：Whisper 支持多语言，中英混合无问题；重方言可能转录质量下降，面后可手动修正
- **面试超时**：无硬性限制，持续到点击结束
- **LLM 响应慢**：追问建议卡片标记"生成中..."，不阻塞其他功能
